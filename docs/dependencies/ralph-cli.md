## Recommended APIs

### Project-scoped long-running config

- Recommended API: `~/.ralph/projects/<project>/config.json`
- Why this is preferred: `Ralph merges global config with per-project config, so repo-specific runtime tuning should live in project storage instead of polluting global defaults.`
- Minimal pattern:

```json
{
  "agent": "codex",
  "maxRetries": 5,
  "agentTimeoutMs": 14400000,
  "stuckThresholdMs": 1800000,
  "maxRuntimeMs": 43200000
}
```

- Notes for this repo: `Use pnpm ralph:bootstrap to write the project-level config into the active Ralph storage directory detected from the current git repo.`

### Detached execution for unattended runs

- Recommended API: `ralph run -b`, `ralph status`, `ralph resume -b`, `ralph stop`
- Why this is preferred: `These are the documented session commands for long-running detached execution, monitoring, resume, and graceful shutdown.`
- Minimal pattern:

```bash
ralph run -b
ralph status
ralph resume -b
ralph stop
```

- Notes for this repo: `Package scripts wrap these commands as pnpm ralph:run:bg, pnpm ralph:status, pnpm ralph:resume:bg, and pnpm ralph:stop.`

### Repo-specific instructions for every iteration

- Recommended API: `~/.ralph/projects/<project>/instructions.md`
- Why this is preferred: `Ralph includes project instructions in every iteration, which is the right place to inject repo conventions, validation habits, and document-reading order.`
- Minimal pattern:

```md
Read the project requirements, topic contract, and active task doc before coding.
Use pnpm and prefer the lightest validation command that proves the edited scope.
```

- Notes for this repo: `The source-of-truth template lives at docs/playbooks/ralph/project-instructions.md and is copied into Ralph storage by the bootstrap script.`

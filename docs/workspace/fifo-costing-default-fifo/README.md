# FIFO 成本核算草案

阶段: 方案草拟
创建: 2026-03-31
最后更新: 2026-03-31

## 当前状况

用户已明确希望系统采用 `FIFO` 作为默认成本消耗规则，目标是实现“入库价可查、出库成本可算”，但暂不追求完整批次模型。当前仓库已有“入库流水 + 来源占用”的基础骨架，但普通出库尚未进入来源分配与成本结转闭环。

当前更合适的方向不是直接上完整批次模型，而是基于现有 `InventoryLog + InventorySourceUsage` 演进为 `FIFO 成本层模型`：保留汇总库存表，补齐入库成本快照、自动 FIFO 分配、出库成本快照、以及反向释放逻辑。

## 待决策项

当前无待用户确认项。本工作流的下一步更适合进入 schema 草案和 service 改造拆解。

## 背景与上下文

- 当前单据行能保存 `unitPrice` / `amount`，因此入库价格本身可追溯。
- 当前 `InventoryBalance` 只保存数量余额，不保存库存金额或平均/分层成本。
- 当前 `InventorySourceUsage` 已支持“某条出库线消耗了哪条入库流水多少数量”，适合作为 FIFO 消耗记录的承载点。
- 当前车间领料支持手工传 `sourceLogId`，说明系统已经接受“来源入库流水”这一语义，但还没有把它提升成全局默认库存消耗规则。

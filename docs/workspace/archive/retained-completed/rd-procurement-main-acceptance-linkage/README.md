## 后续切片释义

### 研发物料独立状态链

这里说的不是“采购单、验收单、领用单本身的单据审核状态”，而是“某一批研发物料在 RD 链路里的业务状态”。

它要回答的是：这批物料现在业务上处在哪一步。例如：

- `待采购`：RD 小仓刚录入采购需求，还没被采购角色正式接单。
- `采购中`：采购侧已经接单，正在外部采购。
- `到货`：实物已经到主仓，但还没有完成主仓到 RD 的真实交接。
- `验收`：这里按已确认口径，指完成主仓到 RD 的交接，不是主仓验收单创建成功。
- `领取`：已经被 RD 项目领用。
- `报废` / `退回` / `取消`：进入对应的后续结局。

这条切片的价值在于把“RD 采购需求 -> 主仓到货 -> 交接到 RD -> 项目领用 / 报废 / 退回”的全过程变成一条可追踪的业务生命线。它重点解决的是“这批 RD 物料现在到哪一步了”，而不是“库存数对不对”。

### 小仓盘点 / 库存调整

这条说的是 RD 小仓自己的账实校准能力。

它要回答的是：如果 RD 小仓现场实物和系统库存对不上，系统怎么把差异合法、可审计地补回来。例如：

- 盘点：按物料核对“系统数量”和“实物数量”，形成盘盈 / 盘亏结果。
- 库存调整：对已经确认的差异做正式调账，让 RD 小仓库存恢复真实值。
- 审计追溯：要知道是谁调的、为什么调、调前调后是多少。

这条切片重点解决的是“RD 小仓库存数字如何被校正”，不是采购链路状态，也不是主仓验收到货联动。

### 为什么要拆成两个后续切片

这两件事容易被混在一起，但本质上不是同一问题：

- “研发物料独立状态链”关注业务流程位置，是流程状态问题。
- “小仓盘点 / 库存调整”关注库存数字校正，是库存治理问题。

前者即使不改一条库存，也可以成立；后者即使完全不关心采购状态，也必须能独立工作。

所以继续做 RD 时，更稳妥的方式是：

- 如果你优先想知道“每批研发物料现在走到哪一步了”，先开“研发物料独立状态链”。
- 如果你优先想解决“RD 小仓库存需要自查、自纠偏”，先开“小仓盘点 / 库存调整”。

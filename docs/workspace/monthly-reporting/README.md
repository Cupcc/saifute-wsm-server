## 当前状况

已识别为跨模块报表能力需求（涉及 reporting、customer、workshop-material、project、scheduler 等模块），并已写入项目级需求。当前仍在等待用户确认月报的具体指标口径和交付方式，尚未进入设计或实施阶段。

## 待确认事项

以下问题需要用户确认后才能进入详细设计：

1. **指标范围**: 月度汇总仅含数量，还是同时包含金额、成本、结存、报废损耗、退料回补？
2. **销售域口径**: 销售月报只看客户收发数量，还是包含销售金额、退货金额、毛利、回款？
3. **生成方式**: 每月定时生成固定快照，还是按月自动汇总但允许实时重算？
4. **交付形式**: 系统页面查看、Excel 导出、PDF、消息推送、邮件发送——需要哪些？
5. **补录处理**: 月末后补录单据，是否允许重算并覆盖已生成月报？

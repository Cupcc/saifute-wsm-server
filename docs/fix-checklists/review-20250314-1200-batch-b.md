## Fix Checklist

- [x] [important] **Workflow getAuditStatus query validation**: Fixed in follow-up. `GET /workflow/audits/status` now validates through `QueryAuditStatusDto`, so missing or empty query values are rejected at the controller boundary instead of flowing through as unvalidated strings.

- [x] [important] **Reverse-stock race condition**: Fixed in follow-up. The re-check records a unique constraint on `inventory_log.reversalOfLogId` plus conflict handling that resolves uniqueness races back to the existing reversal log, closing the duplicate-reversal window noted in the original review.

- [x] [suggestion] **CreateMaterialDto / UpdateMaterialDto precision**: Fixed in follow-up. The re-check records that quantity-like DTO inputs are now validated as decimal strings at the boundary, which aligns these warning-threshold fields more closely with the repo's high-precision quantity handling.

- [x] [suggestion] **Inventory command quantity type**: Fixed in follow-up. `InventoryService` command inputs now accept `string | number | Prisma.Decimal` and normalize internally to `Prisma.Decimal`, reducing float-contract ambiguity at the service boundary.

- [x] [suggestion] **Workflow approve/reject transaction boundary**: Closed by documentation alignment. `docs/modules/workflow.md` explicitly states that `approve` / `reject` / `reset` default to standalone workflow actions, and that business modules needing one atomic transaction must wrap the business write plus `workflow` orchestration in the same application-layer transaction.

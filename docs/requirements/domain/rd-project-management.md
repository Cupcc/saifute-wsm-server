## 主题定义

- `rd-project` / `rdProject` 专指内部研发项目，不包含对外销售项目。
- 本主题解决“研发项目主档如何建模、研发项目 BOM 如何维护、研发项目实际领料 / 退料 / 报废如何发生、项目缺口与成本台账如何查看”的问题。
- 研发项目不是项目管理系统，不承接排期、甘特图、任务协同、审批流或工程进度管理。
- 研发项目不是新的物理库存池；真实库存范围仍只有 `MAIN` 与 `RD_SUB`，研发项目只是内部项目语义与台账维度。
- 本文档保留长期约束、业务口径、能力清单和能力合同；单次交付继续从本 domain 创建 `task-*.md`。

## 承接的项目级能力版图

### 研发项目（`rd-project`）

- 本 domain 承接项目总纲中 `RD / 研发协同与研发项目` 下的内部研发项目职责，作为研发项目主档、BOM、缺料 / 补货视图、项目物料动作和项目台账的真源。
- 研发项目页面需要作为独立入口存在，供小仓管理员或相关人员维护研发项目主档、查看 BOM、查看缺口 / 补货状态、录入项目物料动作，并查看项目层净耗用与成本台账。
- 研发项目的真实库存动作统一通过 `inventory-core` 过账；项目页面不能旁路修改库存。

### 相邻共享边界

- `rd-subwarehouse` 负责研发采购状态链、主仓到研发小仓交接、小仓日常协同与小仓库存责任；本 domain 消费其结果，但不替代该主题。
- `inventory-core` 继续作为所有库存变动的唯一写入口；本 domain 只组织研发项目语义、BOM 和项目动作。
- `sales-project` 是独立的对外销售项目主题；本 domain 不承接销售项目库存视图、销售项目发货统计或客户供货语义。
- `reporting` 负责跨域只读聚合；本 domain 提供稳定的研发项目事实和查询口径。

## 长期约束

- `C1` 研发项目是内部主题：`rd-project` 只承接内部研发项目，不得再与销售项目共用一套定义。状态：`生效中`
- `C2` 固定实际作业仓别：研发项目的实际使用仓别固定为 `RD_SUB`，不得把 `MAIN` 视为研发项目直接使用仓别。状态：`生效中`
- `C3` 不是项目管理系统：本 domain 只承接主档、BOM、缺口 / 补货、项目动作和台账，不引入项目生命周期状态机。状态：`生效中`
- `C4` BOM 不直接改库存：研发项目 BOM 保存、修改、作废都不能直接产生库存副作用。状态：`生效中`
- `C5` 项目物料动作统一过账：研发项目领料、退料、报废等真实库存事实必须通过 `inventory-core` 落账，并保持来源追溯。状态：`生效中`
- `C6` 项目不是库存池：研发项目只承担内部项目维度、成本和查询语义，不形成新的 `stockScope`。状态：`生效中`
- `C7` 采购与主仓交接分域：研发采购状态链、主仓验收和 `RD handoff` 由 `rd-subwarehouse` / `inbound` 承接，研发项目只消费结果。状态：`生效中`
- `C8` 当前命名冻结：当前代码合同、接口、权限、菜单和迁移脚本统一使用 `rd-project` / “研发项目”；不再回退到裸 `project` 或“项目归集”语义。状态：`生效中`

## 长期业务口径

### 项目主档口径

- 研发项目主档至少需要支撑：建档、查询、修改、作废、项目相关 BOM 查看、项目台账查看。
- 主档字段优先围绕内部研发使用场景设计，例如：项目编码、项目名称、业务日期、业务车间、负责人、备注等。
- 研发项目一旦已有关联 BOM、项目物料动作或下游协同事实，应保留历史，不做静默硬删。

### BOM 与缺口口径

- 研发项目 BOM 表达计划物料、计划数量和参考成本，不表达真实库存动作。
- 研发项目缺口 / 补货视图应至少回答：计划量、当前可用、已领、已退、已报废、净耗用、缺口量、补货状态、补货在途。
- 缺口 / 补货视图优先基于 `RD_SUB` 可用库存、项目净耗用结果和 `rd-subwarehouse` 协同结果派生，不另建平行库存账。

### 项目物料动作与台账口径

- 研发项目物料动作是独立业务事实，至少覆盖 `领料 / 退料 / 报废` 三类动作。
- 每个项目动作都必须保留项目、物料、数量、来源关系、操作人、业务日期、作废痕迹和成本结果。
- 项目台账至少应支持：计划数量 / 成本、已领数量 / 成本、已退数量 / 成本、已报废数量 / 成本、净耗用数量 / 成本、缺口量、缺口物料数。

### 查询与角色口径

- RD 小仓管理员需要把研发项目列表和项目详情作为日常入口，用于查看主档、BOM、缺口 / 补货和项目台账。
- 研发项目与销售项目保持完全分域；同一物料可同时存在研发项目使用和销售项目供货视图，但两者不能共享同一项目 ID 语义。

## 能力合同（摘要）

### `F1` 研发项目主档轻量 CRUD

- In scope:
  - 维护项目编码、项目名称、业务日期、业务车间、负责人、备注等主档字段。
  - 提供项目新建、查询、修改、作废和历史保留能力。
- Completion criteria:
  - `[TC-1]` 文档明确研发项目主档不直接改库存。
  - `[TC-2]` 文档明确研发项目只服务内部 RD 主题，不复用销售项目语义。

### `F2` 研发项目 BOM 与缺口 / 补货视图

- In scope:
  - 维护研发项目 BOM 与参考成本。
  - 详情页展示当前可用、缺口量、补货状态和补货在途。
- Completion criteria:
  - `[TC-1]` 文档明确 BOM 写路径不产生库存副作用。
  - `[TC-2]` 文档明确缺口 / 补货视图来自 `RD_SUB` 可用量和协同结果派生。

### `F3` 研发项目物料动作与库存联动

- In scope:
  - 项目领料、退料、报废动作。
  - 统一通过 `inventory-core` 记账并保留来源关系。
- Completion criteria:
  - `[TC-1]` 文档明确项目动作不得旁路 `inventory-core`。
  - `[TC-2]` 文档明确动作类型与来源追溯保持稳定可审计。

### `F4` 研发项目净耗用与成本台账

- In scope:
  - 汇总项目层计划、已领、已退、已报废、净耗用、缺口与成本。
  - 支持项目级台账明细查看。
- Completion criteria:
  - `[TC-1]` 文档明确项目台账不是平行库存账，而是项目维度台账。
  - `[TC-2]` 文档明确净耗用和成本结果来自真实项目动作与来源成本结果。

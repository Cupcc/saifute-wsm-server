# AI 助手主题需求

## Metadata

- ID: `domain-ai-assistant-module`
- Status: `needs-confirmation`
- Scope: `domain-level`

## 主题定义

- `ai-assistant` 是查询与编排辅助主题，解决“用户如何用自然语言查询库存 / 单据 / 预警信息、获得页面导航建议、获得受控的表单预填帮助”的问题。
- 第一阶段 `AI` 助手默认定位为查询、解释、导航和预填辅助工具，不直接写业务数据，不替代正式业务录单。
- 本主题同时承担前端流式交互协议稳定性要求，必须兼容现有 `SSE` 消费方式和页面 action 处理方式。

## 承接的项目级能力版图

### AI 助手（`ai-assistant`）

- 本 domain 承接项目总纲中的 `ai-assistant` 模块详细职责，作为 `AI` 查询、工具编排和前端动作命令的长期真源。
- 第一阶段需要兼容前端现有流式聊天面板，支持查询库存、预警、最近单据、页面入口等典型问答。
- 工具调用默认采用白名单和受控编排，输出结果应以可解释、可回退、不中断主业务为原则。
- `AI` 助手可以依赖 `inventory-core`、`master-data`、`reporting` 及各业务域的公开查询服务，但不越权直查底表，也不拥有新的事务写模型表。

## 长期约束

- `C1` 不直接写业务数据：`AI` 助手只能做查询、建议、导航和预填，不允许直接创建、修改、审核或作废业务单据。状态：`生效中`
- `C2` 工具调用必须白名单化：模型只能调用预先注册的工具，不允许自由执行任意服务方法或数据库查询。状态：`生效中`
- `C3` 协议必须兼容现有前端：`SSE` 事件名、事件顺序和载荷结构必须保持稳定，避免破坏现有聊天面板解析逻辑。状态：`生效中`
- `C4` 权限边界继承原业务域：`AI` 工具返回结果必须服从原业务模块的权限与数据权限，不能因为走了 `AI` 入口就放宽可见范围。状态：`生效中`
- `C5` 页面动作命令需显式建模：页面跳转、打开表单、预填字段等动作必须通过显式命令对象返回，不使用自由文本协议。状态：`生效中`
- `C6` 返回结果必须可解释与可回退：`AI` 输出需要说明依据、来源时间点或可跳转位置，不能产出不可验证的黑盒业务结论。状态：`生效中`
- `C7` 审计优先保留摘要：对话和工具调用建议保留审计摘要，但不强制长期保存完整聊天全文。状态：`生效中`

## 长期业务口径

### 对话与上下文口径

- `AI` 对话默认结合系统提示词、当前页面上下文、用户身份与受权范围组织回答。
- 第一阶段重点支持库存查询、主数据查询、最近单据查询、报表入口导航和常见业务概念解释。
- `AI` 对话的主要目标是帮助用户更快找到信息和操作入口，而不是替用户完成最终业务决策。

### 工具与权限口径

- 工具执行层只调用各模块公开的查询服务，不越权直查底表。
- 工具输出必须遵守登录态、角色权限、菜单权限和数据权限约束。
- 当用户请求超出权限或超出白名单工具范围时，`AI` 应明确说明限制，而不是编造结果或假装成功执行。

### 导航与预填口径

- 页面跳转命令用于引导用户前往正确页面或详情入口，不直接代表业务已执行。
- 表单预填命令用于帮助用户减少重复录入，但最终提交仍由用户在正式业务页面完成。
- 导航与预填契约需要可版本化，避免前后端协议演进时互相漂移。

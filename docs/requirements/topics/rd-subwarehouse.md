## 主题定义

- `RD 小仓` 是项目/研发域下的长期业务主题，解决“主仓 + 研发小仓如何在同一系统内协同作业”的问题。
- `RD 小仓` 在业务上按真实独立运转的小仓处理，不是简单的研发领用标签、项目视图或报表过滤条件。
- 本文档保留长期约束、长期业务口径、能力清单和阶段路线图；单次交付直接从本 topic 能力合同创建 `task-*.md`，执行细节继续放在 `docs/tasks/*.md`。

## 长期约束

- `C1` 同平台协同：主仓与研发小仓在同一软件平台内协同作业，不拆成两套系统。状态：`生效中`
- `C2` 真实独立小仓：`RD 小仓` 按真实独立运转的小仓建模，而不是附着在主仓之上的标签化视图。状态：`生效中`
- `C3` 受限子仓模型：当前按“主仓 + 研发小仓受限协同”处理，不直接扩展为完整开放式多仓体系。状态：`生效中`
- `C4` 展示与壳层策略：采用折中隔离和非镜像双视角；RD 侧以独立工作台、本仓数据和本仓报表为主，同时允许少量公共能力保留；系统管理员保持双侧全量可见，不受专属壳层裁切影响。状态：`生效中`
- `C5` 公司级流转边界：货物进入公司、离开公司及公司级退回/退货仍统一经过主仓链路处理，RD 小仓不直接承接外部采购入库、外部退回入库或对外退货。状态：`生效中`
- `C6` 库存真源：所有库存写入统一经过 `inventory-core`，任何页面或单据都不能旁路改库存。状态：`生效中`
- `C7` 项目归集强绑定：RD 小仓出库与 RD 采购物料必须关联研发项目或项目式归集项，不能脱离项目单独流转。状态：`生效中`
- `C8` 角色边界：小仓管理员默认只处理本仓日常作业；公司级外部入出、退回与跨域排障仍由主仓侧或系统管理员处理。状态：`生效中`
- `C9` 实物库存与归属分离：第一阶段真实库存范围只有主仓与 RD 小仓；生产车间不是独立库存池，`workshop` 只承担主仓领退料归属与成本核算。状态：`生效中`

## 长期业务口径

### 角色与展示口径

- 大仓管理员以主仓运营面和必要的 RD 协同入口为主，可查看与研发协同有关的记录，但不进入 RD 专属日常操作台，也不承担小仓日常盘点、报废、领用执行。
- 小仓管理员默认进入 `研发工作台`，只关注本小仓库存、流水、报表、项目归集信息和主仓协同结果，不看主仓运营面。
- 系统管理员同时可见主仓侧与 RD 小仓侧入口及页面，用于统一配置、排障与跨域查看；这不是第三套业务流，而是保留双侧可见与系统级管控能力。

### 协同与库存责任口径

- 主仓发料或调拨到 RD 小仓后，系统自动完成到小仓的库存过账，小仓无需二次收货确认。
- 研发采购实物到达主仓且已填写验收单后，先由主仓验收入主仓；只有在实际完成主仓到小仓交接时，库存才从主仓转入小仓。
- RD 小仓当前阶段至少承接：查看本仓库存与流水、查看主仓调拨后自动入小仓结果、执行本仓出库/领用、执行盘点/库存调整、执行本仓报废、查看仅属于本仓的报表。
- 生产车间领料、退料继续以主仓为真实库存出入账范围；系统只按 `workshop` 记录归属车间、月度净耗用与成本，不统计车间在手库存。
- 小仓管理员不负责退回主仓或对外退货等公司级二次流转动作，相关退料、退回与退货仍由主仓统一处理。

### 项目归集与采购口径

- RD 小仓出库原则上必须关联研发项目；办公用品等通用消耗也必须落到明确的项目式归集项。
- RD 小仓可以提出采购需求，由小仓管理员录入采购信息，再由采购角色执行采购。
- 主仓需要可见 RD 采购信息，以便对照到货实物完成验收核对。
- 主仓填写验收单时应能直接选择对应 RD 采购信息并自动带出内容，避免重复手工填写。

### 物料状态口径

- RD 采购链路中的物料需要具备独立状态跟踪，该状态不等同于采购单、验收单或领料单自身的审核状态。
- 状态链至少包括 `待采购 / 采购中 / 取消 / 验收 / 领取 / 报废 / 退回`。
- 其中 `验收` 明确指实物到达主仓且已填写验收单，`领取` 明确指完成主仓到小仓交接，`退回` 属于公司级流转结果并仍由主仓统一执行。
